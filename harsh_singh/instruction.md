## 1. Project Overview

You are a web developer tasked with creating a responsive online gaming website for ArcadeNest Play, a small gaming brand focused on casual browser game discovery.

The website must help visitors browse sample game cards, view game details, read community updates, check leaderboard previews, and send a support message through a contact form layout. This is a Web Development task with a gaming theme. It is not a Game Development task.

The final website must be a front end project that runs locally in a browser. The project may use HTML, CSS, and JavaScript with no build step, or it may use a front end framework if setup commands and run commands are included.

The seller must not build playable games, account systems, payment systems, real time multiplayer systems, gambling features, casino features, prize money flows, or a back end database.

## 3. Required Website Pages

### 3.1 Overview

The website must include six main pages or sections:

| Page | Required Status | Purpose |
|---|---|---|
| Homepage | Required | Introduces ArcadeNest Play and guides visitors to games |
| Game Library | Required | Lets visitors search and filter sample games |
| Game Detail | Required | Shows one game description, controls, rating, and related games |
| Leaderboard | Required | Shows ranked sample scores |
| Community Updates | Required | Shows news and update cards |
| Support | Required | Shows contact form layout and FAQ content |

### 3.2 Homepage Requirements

The homepage must include these sections in this order:

| Section | Minimum Requirement |
|---|---|
| Header | Logo text and navigation links |
| Hero | One headline, one supporting line, one primary button, one secondary button |
| Featured Games | At least four game cards |
| Game Categories | At least four category tiles |
| Leaderboard Preview | At least three score rows |
| Community Preview | At least three update cards |
| Footer | Navigation links, support link, brand note, copyright line |

Each homepage game card must show a game title, category, difficulty label, short description, rating value, and action button.

### 3.3 Game Library Requirements

The Game Library page must include:

- Page title.
- Search input for filtering game titles.
- Category filter buttons for Arcade, Puzzle, Racing, Strategy, and Action.
- At least eight game cards.
- Empty state text that appears when no games match the search.
- A visible action button on every game card.

Each game card must show game title, category, difficulty label, short description, rating value, and action button.

### 3.4 Page-Specific Requirements

The Game Detail page must include game title, large preview panel, description, controls or rules section, difficulty label, rating value, play button placeholder, and at least three related game cards. The play button must not launch a real game.

The Leaderboard page must include Daily, Weekly, and Monthly tabs with at least ten rows. Each row must show rank, player name, game name, score, and time period. The top three ranks must have a distinct visual treatment.

The Community Updates page must include at least six update cards, with title, date, category tag, and one paragraph summary. It must also include a newsletter signup layout with email field and submit button.

The Support page must include support intro text, contact form layout, name field, email placeholder, message field, submit button, FAQ section with at least five questions, support response time text, and footer repeated from the homepage.

---

## 4. Navigation and Interaction Requirements

The header navigation must include Home, Games, Leaderboard, Community, and Support. Navigation links must move the user to the matching page or section. The active page must be visually marked.

The footer must include Home, Games, Leaderboard, Community, Support, Terms, and Privacy. Terms and Privacy may route to placeholder sections if separate pages are not created.

Required interactions:

- Game search filters game cards by title.
- Category filters filter game cards by Arcade, Puzzle, Racing, Strategy, or Action.
- Leaderboard tabs update visible leaderboard rows.
- Support form validates that name, email, and message fields are not empty.
- Newsletter form validates that the email input contains an at sign.
- Hover or focus states must be visible on buttons and interactive cards.

---

## 5. Visual and Responsive Requirements

Body text must be at least 16 px. Primary buttons must be at least 44 px tall. Text contrast must be at least 4.5 to 1 for body text against its background. Icons may support labels, but icons must not replace text labels for navigation, forms, buttons, or filters. All images must include alt text. Form inputs must have labels.

| Screen Type | Width Range | Game Card Columns |
|---|---|---|
| Wide desktop | 1200 px and above | 4 columns |
| Standard desktop | 1024 px to 1199 px | 3 columns |
| Tablet | 768 px to 1023 px | 2 columns |
| Mobile | 360 px to 767 px | 1 column |

No horizontal scrolling may appear at 360 px width. Header navigation may collapse into a menu button on mobile. The mobile menu must expose every required navigation link.
